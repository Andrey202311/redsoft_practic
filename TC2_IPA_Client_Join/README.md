## 1. Предварительные условия

1. Выполнен Тест-кейс 1 (сервер IPA установлен и настроен на ВМ1, IP: 192.168.3.10, FQDN: server.redosipa.test)
2. ВМ2 (client1) установлена и обновлена

4. Имя хоста: **client1.redosipa.test**
5. Синхронизация времени настроена (chrony)
6. ВМ1 (192.168.3.10) и ВМ2 (192.168.3.11) видят друг друга по IP (ping проходит)
7. Снимок ВМ2 создан перед началом тестирования
8. Доступ к учетной записи root на ВМ2

## 2. Процедура тестирования

### Шаг 2.1: Проверка текущего имени хоста

**Действие:** Выполнить:
```bash
# hostnamectl
```

**Ожидаемый результат:**
- Static hostname: `client1.redosipa.test`
- Если имя не совпадает — установить: `hostnamectl set-hostname client1.redosipa.test`

---

### Шаг 2.2: Отключение DNS-службы systemd-resolved на клиенте

**Действие:**
1. Открыть файл `/etc/systemd/resolved.conf`:
```bash
# nano /etc/systemd/resolved.conf
```
2. Установить `DNSStubListener=no`
3. Сохранить и выйти
4. Перезапустить службу:
```bash
# systemctl restart systemd-resolved.service
```

**Ожидаемый результат:** Служба перезапущена без ошибок. Порт 53 освобождён.

---

### Шаг 2.3: Настройка сетевого адаптера (DNS)

**Действие:**
1. Настроить DNS в графическом интерфейсе или через конфигурационные файлы:
   - **Дополнительные серверы DNS**: `192.168.3.10` (IP ВМ1 — server)
   - **Дополнительные поисковые домены**: `redosipa.test`
2. Перезапустить сетевые службы:
```bash
# systemctl restart NetworkManager
```
3. Проверить конфигурацию DNS:
```bash
# cat /etc/resolv.conf
```
4. Проверить SRV-запись:
```bash
# dig SRV _ldap._tcp.redosipa.test
```

**Ожидаемый результат:**
- В `/etc/resolv.conf` указан nameserver `192.168.3.10`
- `dig SRV` возвращает ответ от сервера ВМ1 (192.168.3.10)
- Запросы DNS резолвятся через IPA-сервер

---

### Шаг 2.4: Установка необходимых пакетов

**Действие:** Выполнить на ВМ2:
```bash
# dnf install ipa-client
```

**Ожидаемый результат:** Пакет `ipa-client` установлен успешно с зависимостями.

---

### Шаг 2.5: Интерактивная настройка клиента (способ 1 — рекомендуется)

**Действие:**
1. Запустить интерактивную настройку:
```bash
# ipa-client-install --mkhomedir
```
2. Подтвердить обнаруженные значения: `yes`
3. Ввести учетные данные администратора домена:
   - User authorized to enroll computers: `admin`
   - Password: `<пароль_администратора_IPA>`

**Ожидаемый результат:**
- Утилита автоматически обнаруживает сервер (192.168.3.10), домен и realm
- Выводится сообщение: `Client configuration complete. The ipa-client-install command was successful`

> **Примечание:** Лог установки: `/var/log/ipaclient-install.log`

---

### Шаг 2.6: Проверка ввода в домен (CLI)

**Действие:**
1. Получить билет Kerberos:
```bash
# kinit admin
```
2. Проверить, что клиент зарегистрирован:
```bash
# ipa host-find client1.redosipa.test
```

**Ожидаемый результат:**
- `kinit` выполняется успешно
- Клиент отображается в списке хостов IPA-сервера

---

### Шаг 2.7: Ввод в домен через графическую утилиту join-to-domain (способ 2)

**Действие:**
1. Обновить утилиту:
```bash
# dnf update join-to-domain
# dnf install sssd-kcm
```
2. Запустить утилиту: Главное меню → Система → Ввод ПК в домен
3. Выбрать "Домен IPA"
4. Заполнить поля:
   - Имя домена: `redosipa.test`
   - Имя компьютера: `client1`
   - Имя администратора: `admin`
   - Пароль администратора: `<пароль>`
5. Нажать "Сохранить"

**Ожидаемый результат:**
- Выводится сообщение: "Компьютер успешно введен в домен IPA! Перезагрузите ПК"
- После перезагрузки клиент работает в домене

---

### Шаг 2.8: Ввод в домен через терминал join-to-domain (способ 3)

**Действие:**
1. Запустить утилиту в терминале:
```bash
# join-to-domain
```
2. Выбрать тип домена: `2` (Домен IPA)
3. Подтвердить имя домена
4. Ввести имя ПК: `client1`
5. Ввести имя администратора: `admin`
6. Ввести пароль администратора
7. Подтвердить продолжение: `y`

**Ожидаемый результат:**
- Выводится сообщение: "РЕД ОС успешно введён в домен IPA! Перезагрузите ПК."
