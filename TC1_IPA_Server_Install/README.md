## 1. Предварительные условия

1. ВМ1 (server) установлена и обновлена (выполнено `dnf up -y`)
2. Статический IP-адрес назначен: **192.168.3.10** (не изменяется)
3. Имя хоста: **server.redosipa.test**
4. Синхронизация времени настроена (chrony)
5. Порт 53 освобождён (systemd-resolved настроен)
6. Снимок ВМ1 создан перед началом тестирования
7. Доступ к учетной записи root

---

## 2. Процедура тестирования

### Шаг 2.1: Проверка текущего имени хоста

**Действие:** Выполнить:
```bash
# hostnamectl
```

**Ожидаемый результат:**
- Static hostname: `server.redosipa.test`
- Если имя не совпадает — установить: `hostnamectl hostname server.redosipa.test`

---

### Шаг 2.2: Отключение DNS-службы systemd-resolved

**Действие:**
1. Открыть файл `/etc/systemd/resolved.conf` для редактирования:
```bash
# nano /etc/systemd/resolved.conf
```
2. Установить параметр `DNSStubListener=no`
3. Сохранить файл и выйти (Ctrl+O, Enter, Ctrl+X)
4. Перезапустить службу:
```bash
# systemctl restart systemd-resolved.service
```

**Ожидаемый результат:** Служба перезапущена без ошибок. Порт 53 освобождён.

---

### Шаг 2.3: Установка необходимых пакетов

**Действие:** Выполнить установку:
```bash
# dnf install ipa-server-dns pki-ca pki-kra pki-acme
```

**Ожидаемый результат:** Пакеты установлены успешно. Автоматически установлены зависимости (389-ds-base, krb5-server и др.).

---

### Шаг 2.4: Настройка сервера в интерактивном режиме

**Действие:**
1. Запустить интерактивную установку:
```bash
# ipa-server-install --ip-address=192.168.3.10
```
2. Подтвердить настройку интегрированного DNS: `yes`
3. Подтвердить или изменить имя хоста, домен и realm:
   - Server host name: `server.redosipa.test` (подтвердить Enter)
   - Domain name: `redosipa.test` (подтвердить Enter)
   - Realm name: `REDOSIPA.TEST` (подтвердить Enter)
4. Задать пароль Directory Manager (DM) — минимум 8 символов
5. Задать пароль администратора IPA
6. DNS forwarders: `no` (из-за таймаутов внешнего DNS)
7. Создать обратную зону: `yes`
8. Подтвердить конфигурацию: `yes`

**Ожидаемый результат:**
- Установка завершается сообщением `The ipa-server-install command was successful`
- Выводятся рекомендации по открытию портов и созданию резервной копии сертификата ЦС
- Создан файл `/root/cacert.p12`

> **Примечание:** Лог установки: `/var/log/ipaserver-install.log`

---

### Шаг 2.5: Проверка работоспособности сервера

**Действие:**
1. Получить билет Kerberos:
```bash
# kinit admin
```
2. Ввести пароль администратора IPA
3. Выполнить поиск пользователя:
```bash
# ipa user-find admin
```

**Ожидаемый результат:**
- Команда `kinit` выполняется без ошибок
- Команда `ipa user-find admin` возвращает информацию о пользователе admin:
  - Имя учётной записи: admin
  - Фамилия: Administrator
  - UID: 1479400000 (фактическое значение при данной установке)
  - Учётная запись отключена: False

---

### Шаг 2.6: Настройка брандмауэра

**Действие:**
1. Установить firewalld:
```bash
# dnf install firewalld
```
2. Запустить и добавить в автозагрузку:
```bash
# systemctl enable --now firewalld
```
3. Открыть порты:
```bash
# firewall-cmd --permanent --add-port=53/{tcp,udp} --add-port={88,464}/{tcp,udp} --add-port=123/udp --add-port={389,636}/tcp
# firewall-cmd --add-service={http,https,ldap,ldaps} --permanent
# firewall-cmd --reload
```

**Ожидаемый результат:**
- firewalld установлен и запущен
- Правила применены без ошибок
- Команда `firewall-cmd --list-all` показывает открытые порты и сервисы
